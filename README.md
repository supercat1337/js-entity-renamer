# JS Entity Renamer 🔄

A command-line tool for **bulk renaming** of JavaScript entities (variables, functions, classes, methods, properties, including private fields) using a mapping file. It parses your code with Babel, replaces all occurrences of specified identifiers, and outputs the transformed code while preserving comments and formatting.

⚠️ **Warning**: This is a "dumb" renamer – it replaces all instances of an identifier regardless of scope. It does not check for naming collisions or semantic correctness. Use with caution and always review the results.

## ✨ Features

- Renames **variables, functions, classes, methods, properties, and private fields** (`#privateField`).
- Works with modern JavaScript (ES2022+), including optional chaining, nullish coalescing, and class private features.
- Preserves comments and original code formatting.
- Supports glob patterns for batch processing.
- Dry-run mode to preview changes without writing files.
- Map file can be generated by [js-entity-extractor](https://github.com/supercat1337/js-entity-extractor) (or written manually).

## 📦 Installation

### Global install (recommended)

```bash
npm install -g https://github.com/supercat1337/js-entity-renamer
```

### Local install (for npm scripts)

```bash
npm install --save-dev js-entity-renamer
cd js-entity-renamer
npm link
```

### Run without installing

```bash
npx js-entity-renamer [options] <patterns...>
```

## 🚀 Usage

```bash
js-entity-renamer [options] <patterns...>
```

### Basic examples

```bash
# Rename identifiers in a single file using map.json
js-entity-renamer src/app.js --map map.json

# Process all .js files recursively and write results to a separate directory
js-entity-renamer "src/**/*.js" --map map.json --output-dir ./renamed

# Exclude test files and perform a dry run
js-entity-renamer . -r --map map.json --exclude "**/*.test.js" --dry-run

# Use a map generated by js-entity-extractor
js-entity-extractor "src/**/*.js" --dictionary-map map.json   # create map (keys = original names)
# (edit map.json manually, replace values with desired new names)
js-entity-renamer "src/**/*.js" --map map.json -o ./renamed
```

## ⚙️ Options

| Option                   | Description                                                           |
| ------------------------ | --------------------------------------------------------------------- |
| `-m, --map <file>`       | **Required.** Path to the JSON mapping file (see format below).       |
| `-o, --output-dir <dir>` | Output directory. If omitted, input files are overwritten.            |
| `-r, --recursive`        | Process directories recursively (adds `/**/*.js` to directory paths). |
| `--exclude <pattern>`    | Exclude files/directories matching glob (can be repeated).            |
| `--dry-run`              | Show what would be changed without writing files.                     |
| `-h, --help`             | Show help.                                                            |

## 📝 Map File Format

The mapping file must be a JSON object where keys are the **original names** and values are the **new names**.  
For private fields (e.g., `#allValid`), the key must include the `#` symbol.

```json
{
    "oldVariable": "newVariable",
    "oldFunction": "newFunction",
    "#privateField": "#renamedPrivateField",
    "ClassName": "NewClassName"
}
```

> 💡 You can generate a starter map with all names using [js-entity-extractor](https://github.com/supercat1337/js-entity-extractor) and the `--dictionary-map` option.

## ⚠️ Limitations & Caveats

- **Scope‑unaware** – If two different variables in different scopes share the same name, both will be renamed.
- **No collision detection** – If a new name already exists in the same scope, the resulting code may break.
- **Property access** – Object properties like `obj.oldName` are also renamed (including those in destructuring). This is intentional but may affect external APIs.
- **Strings and comments** are never touched.
- Always run with `--dry-run` first and review the changes.

## 🔧 How It Works

1. Parses each JavaScript file using `@babel/parser`.
2. Traverses the AST and replaces any identifier (including private ones) that matches a key in the map.
3. Generates new code with `@babel/generator`, preserving comments and formatting.
4. Writes the transformed file (either in‑place or to the output directory).
